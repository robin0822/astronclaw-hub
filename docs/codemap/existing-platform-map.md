# 现有平台路由地图

## 前端入口

现有演示系统是一个前端单页应用，入口为：

- `/ops`

页面标题：

- 讯飞 AstronClaw 智能体管理平台

## 一级路由

| 路由 | 模块 | 本次功能关系 |
| --- | --- | --- |
| `/agents` | 智能体管理 | 智能体龙虾主入口。 |
| `/org` | 组织权限 | 创建、查看、操作龙虾时需要做 RBAC 和数据范围校验。 |
| `/monitoring` | 监控告警 | 龙虾运行状态、指标和告警来源。 |
| `/models` | 模型管理 | 创建龙虾时选择主模型、备用模型和模型策略。 |
| `/ops` | 运维自动化 | 部署、重启、日志清理、备份恢复等运行操作。 |
| `/skills` | Skill 管理 | 龙虾绑定 Skill 的来源。 |
| `/knowledge` | 知识库 | 龙虾绑定知识库的来源。 |
| `/memory` | 记忆管理 | 龙虾记忆预览、共享和开关能力来源。 |
| `/seats` | 席位管理 | 创建、授权、共享时做席位校验。 |
| `/channels` | 消息渠道 | P0 不完整接入，仅预留业务嵌入和推送能力。 |
| `/security` | 安全审计 | 操作审计、安全策略和导出审计。 |
| `/diagnosis` | 问题诊断 | P0 可只跳转或预留；后续接异常根因和一键修复。 |
| `/architecture` | 技术架构 | 展示平台架构说明，不承载业务操作。 |
| `/share/:id` | 分享页 | 龙虾分享链接入口。 |

## 本次功能落点

智能体龙虾管理模块优先改造 `/agents`，并通过后端聚合关联模块数据。

不要在本次需求中新增独立一级菜单，除非产品确认“龙虾”需要从“智能体管理”中拆成单独业务域。
