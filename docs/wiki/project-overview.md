# AstronClaw 平台项目总览

## 项目定位

AstronClaw 是面向企业私有化场景的智能体管理平台，用于统一管理智能体实例、模型、Skill、知识库、记忆、监控告警、运维诊断和安全审计。

平台已有演示入口：

- 运营管理入口：`https://astronclaw-demo.xfyun.cn/ops`

## 目标用户

- 平台管理员：负责账号、权限、模型、系统配置和安全审计。
- 运营人员：负责智能体实例日常管理、状态查看、启停操作和问题跟进。
- 运维人员：负责监控告警、远程运维、问题诊断和故障闭环。
- 业务负责人：关注智能体使用情况、资源占用、成本和服务质量。

## 已有平台模块

现有演示平台已经具备以下一级模块：

| 模块 | 路由 | 说明 |
| --- | --- | --- |
| 智能体管理 | `/agents` | 管理智能体/龙虾实例的列表、详情、创建、生命周期操作。 |
| 组织权限 | `/org` | 组织架构、角色权限、数据范围。 |
| 监控告警 | `/monitoring` | 运行指标、告警规则、告警闭环。 |
| 模型管理 | `/models` | 模型台账、密钥引用、限流与路由策略。 |
| 运维自动化 | `/ops` | 巡检、自愈、备份、远程操作。 |
| Skill 管理 | `/skills` | Skill 导入、审核、版本和绑定。 |
| 知识库 | `/knowledge` | 知识库、文件解析、向量化、权限。 |
| 记忆管理 | `/memory` | 会话、个人、组织、企业记忆。 |
| 席位管理 | `/seats` | 席位包、分配、回收和预警。 |
| 消息渠道 | `/channels` | 渠道台账、业务系统嵌入和推送预留。 |
| 安全审计 | `/security` | 审计日志、安全策略、导出审计。 |
| 问题诊断 | `/diagnosis` | 异常实例、告警根因、一键修复建议。 |
| 技术架构 | `/architecture` | 平台分层架构和技术选型说明。 |

## 当前功能迭代

当前迭代不按 0-1 新项目处理，而是在已有平台中新增或完善：

- 需求编号：`LOBSTER-0001`
- 需求名称：智能体龙虾管理模块
- 主入口：`/agents`
- 关联模块：`/skills`、`/knowledge`、`/memory`、`/models`、`/monitoring`、`/ops`、`/security`

## 系统边界

前端只访问业务后端 `/api/v1/astron-claw`，不直接访问 Claw Proxy。

业务后端负责鉴权、权限、审计、数据持久化、任务编排和错误码转换；外部沙箱能力通过适配层调用 `/api/v1/bot`。

Claw Proxy 只作为运行时和沙箱能力提供方，不承担平台业务权限、审计和管理台聚合职责。
