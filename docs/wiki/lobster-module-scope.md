# 智能体龙虾模块范围

## 业务定义

“龙虾”是 AstronClaw 平台中可被管理、部署、启停、绑定能力并承载业务服务的智能体实例。

它不是单纯的配置表单，而是一个同时包含基础配置、运行状态、模型配置、Skill 绑定、知识库绑定、日志、告警和审计记录的业务对象。

## P0 范围

第一阶段优先做出龙虾实例管理的最小闭环：

- 龙虾列表：支持状态、名称、部门、负责人、模型等基础筛选。
- 龙虾详情：展示基础配置、运行状态、模型、Skill、知识库、日志、审计。
- 创建龙虾：完成权限、模型、Skill、知识库、席位等前置校验。
- 部署龙虾：创建异步部署任务，成功后回写运行实例 ID。
- 生命周期操作：启动、停止、重启、删除。
- Skill 绑定：支持查看、绑定、解绑已审核 Skill。
- 知识库绑定：支持查看、绑定、解绑有权限的知识库。
- 状态同步：周期性同步运行状态、资源指标和服务状态。
- 基础日志：支持按实例查询运行日志、部署日志、操作日志。
- 操作审计：所有高影响操作必须留痕。

## 暂不纳入 P0

以下能力先预留数据模型或接口，不阻塞 P0 验收：

- 批量部署和批量升级。
- 灰度升级和版本回滚。
- 复杂审批流。
- 成本核算和预算告警。
- 消息渠道完整接入。
- Agent Team。
- 自动诊断和自动自愈。
- 多租户资源隔离的完整策略。

## 生命周期状态

龙虾实例建议使用以下状态：

| 状态 | 含义 |
| --- | --- |
| `draft` | 已创建但尚未部署。 |
| `deploying` | 部署任务执行中。 |
| `running` | 运行中，可对外服务。 |
| `stopped` | 已停止，不对外服务。 |
| `abnormal` | 运行异常，需要排查或修复。 |
| `archived` | 已归档，不再参与常规管理。 |
| `deleted` | 已软删除，仅保留审计记录。 |

## P0 验收标准

- 管理员可以创建一只龙虾，并完成部署任务。
- 列表和详情可以展示基础配置、运行状态、模型、Skill、知识库。
- 启动、停止、重启、删除操作有明确状态变化和审计记录。
- 部署失败、Claw Proxy 异常、权限不足时返回可理解的业务错误。
- 前端不直接调用 Claw Proxy。
- 所有敏感字段不明文返回给前端。
