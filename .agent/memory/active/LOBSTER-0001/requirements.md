# 需求澄清

## 目标

在已有 AstronClaw ops 平台中完成“智能体龙虾”管理模块 P0 建设，使平台管理员、运营人员和运维人员能够围绕一个龙虾实例完成创建、部署、运行态查看、启停重启、删除、Skill/知识库绑定、日志查看和操作审计的最小闭环。

本需求不建设新的独立产品，也不新增独立一级菜单。功能主入口沿用智能体管理 `/agents`，并通过业务后端 `/api/v1/astron-claw` 聚合模型、Skill、知识库、监控、日志、权限和审计数据。前端不直接访问 Claw Proxy，所有沙箱运行时能力由业务后端适配 `/api/v1/bot`。

## 本次范围

- 龙虾列表：
  - 在 `/agents` 展示龙虾实例列表。
  - 支持按状态、名称、部门、负责人、模型筛选。
  - 列表展示实例名称、运行实例 ID、状态、版本、部门、负责人、容器数、Skill 数、绑定知识库、主模型、备用模型、资源配置、当前服务人数、QPS、创建时间、最近更新时间等 P0 字段。
- 龙虾详情：
  - 在 `/agents/:id` 展示基础配置、运行状态、模型配置、Skill、知识库、部署历史、基础日志、告警摘要和操作审计。
  - 运行态数据以最近一次同步快照为准，并展示同步时间和数据来源。
- 创建龙虾：
  - 支持填写名称、描述、部门、负责人、主模型、备用模型、资源配置、可选 Skill、可选知识库。
  - 最小必填字段为名称、部门、负责人、主模型、资源配置；描述、备用模型、Skill、知识库为可选项。
  - 创建前完成用户权限、数据范围、席位、模型状态、资源配额、Skill 权限和知识库权限校验。
  - 创建成功后保存为 `draft` 或按产品确认直接进入 `deploying`。
- 部署龙虾：
  - 创建异步部署任务，后端 worker 调用 Claw Proxy 部署能力。
  - Claw Proxy `/api/v1/bot` 的部署、启停、重启、Skill 安装、状态查询接口契约和错误码已有可联调版本，P0 按正式契约联调。
  - 部署成功后回写 `proxy_instance_id`，状态流转为 `running`。
  - 部署失败时记录失败阶段、错误码和可理解的业务错误，状态流转为 `abnormal` 或 `stopped`。
- 生命周期操作：
  - 支持启动、停止、重启、删除。
  - 所有生命周期操作必须校验当前状态、操作权限和数据范围。
  - 删除采用软删除，保留审计、状态事件和历史记录。
  - `running` 状态不允许直接删除，也不允许自动执行“停止后软删除”；用户必须先手动停止，状态变为 `stopped` 后才能删除。
- Skill 绑定：
  - 支持查看、绑定、解绑已审核且有权限的 Skill。
  - 绑定或解绑运行中龙虾时，后端负责调用 Claw Proxy 安装或卸载能力，并记录本地绑定关系。
- 知识库绑定：
  - 支持查看、绑定、解绑当前用户有权限访问的知识库。
  - 删除知识库或取消授权时，需要能识别是否存在龙虾引用。
- 状态同步：
  - 后端周期性同步运行状态、资源指标、服务状态和外部实例可用性。
  - 同步结果写入运行状态快照，列表和详情读取快照聚合结果。
- 基础日志：
  - 支持按实例查询运行日志、部署日志和操作日志。
  - 敏感字段必须脱敏，查看或导出敏感日志需写审计。
- 操作审计：
  - 创建、编辑、部署、启动、停止、重启、删除、绑定/解绑 Skill、绑定/解绑知识库、修改模型配置等高影响操作必须写审计日志。
- API 与数据模型：
  - API 统一使用 `/api/v1/astron-claw` 前缀。
  - P0 最小表集合包括 `agents`、`agent_versions`、`agent_deploy_tasks`、`agent_runtime_snapshots`、`agent_runtime_configs`、`agent_bind_skills`、`agent_bind_knowledge`、`agent_state_events`、`agent_logs_index`、`audit_logs`。

## 用户与业务规则

- 用户角色：
  - 平台管理员：可查看和管理全部或授权范围内的龙虾、模型、Skill、知识库、审计记录。
  - 运营人员：可在授权数据范围内查看、创建、部署和维护龙虾。
  - 运维人员：可查看运行状态、日志、告警，并执行授权范围内的启停、重启和问题处理操作。
  - 业务负责人：可查看负责范围内的龙虾运行情况、资源占用和服务质量摘要。
  - 系统任务：负责部署、状态同步、日志索引、告警写入等自动化动作。
- 权限规则：
  - 所有读写操作必须经过登录态、角色权限和数据范围校验。
  - 建议权限项至少包括 `agent:view`、`agent:create`、`agent:update`、`agent:delete`、`agent:deploy`、`agent:audit_view`。
  - 用户只能选择自己有权限的部门、模型、Skill 和知识库。
  - 前端不得根据隐藏按钮替代后端权限校验。
- 状态规则：
  - 龙虾生命周期状态使用 `draft`、`deploying`、`running`、`stopped`、`abnormal`、`archived`、`deleted`。
  - 每次状态变化必须写入 `agent_state_events`，记录变化前状态、变化后状态、原因、操作人或任务 ID。
  - `deploying` 状态下不允许重复部署、删除或并发修改关键配置。
  - `running` 状态下执行删除必须返回状态不允许的业务错误，并提示先停止实例。
  - `deleted` 为软删除终态，常规列表默认不展示。
- 创建规则：
  - 名称在同一部门或同一租户范围内是否唯一待确认；未确认前建议按同一部门唯一处理。
  - 最小必填字段为名称、部门、负责人、主模型、资源配置。
  - 主模型必填，备用模型可选。
  - 模型密钥使用后端密钥系统引用，前端不能传入或读取完整密钥。
  - Skill 和知识库可在创建时绑定，也可创建后再绑定。
- 部署规则：
  - 部署是异步任务，前端通过任务状态或详情轮询展示进度。
  - 后端生成业务侧 `bot_id`，调用 Claw Proxy 后保存返回的 `proxy_instance_id`。
  - 部署失败必须记录失败阶段、外部错误码、转换后的业务错误码和可读错误信息。
- 绑定规则：
  - 未审核、禁用、无权限或安全扫描未通过的 Skill 不允许绑定。
  - 无权限、解析失败、已删除或被停用的知识库不允许绑定。
  - Skill/知识库绑定关系变化必须写审计。
- 日志与审计规则：
  - 基础日志按 `agent_id`、`proxy_instance_id`、日志类型和时间范围查询。
  - 审计日志保留周期默认不少于 180 天，最终以客户合规要求为准。
  - 审计和日志中的密钥、令牌、个人敏感信息、模型输入输出敏感片段必须脱敏。
- 外部系统规则：
  - 前端只访问 `/api/v1/astron-claw`。
  - 业务后端负责把 Claw Proxy、监控系统、日志系统等外部错误转换为平台业务错误。
  - P0 验收要求真实接入监控/日志系统；开发环境 mock 数据只作为联调兜底，响应必须标记 `dataSource=mock`，且不能替代 P0 验收依据。

## 验收标准

- [ ] Given 管理员拥有 `agent:create` 权限和可用席位，When 创建一只填写必填字段的龙虾，Then 系统保存龙虾基础配置，并返回平台内部龙虾 ID 和初始状态。
- [ ] Given 用户没有目标部门数据权限，When 创建或查看该部门龙虾，Then 后端拒绝请求并返回权限不足错误，同时不泄露龙虾敏感信息。
- [ ] Given 龙虾处于可部署状态，When 用户触发部署，Then 系统创建部署任务，状态进入 `deploying`，部署成功后回写 `proxy_instance_id` 并流转为 `running`。
- [ ] Given Claw Proxy 部署失败，When 部署任务结束，Then 系统记录失败阶段和外部错误码，转换为前端可理解的业务错误，并将龙虾标记为 `abnormal` 或 `stopped`。
- [ ] Given 列表存在多只龙虾，When 用户按状态、名称、部门、负责人、模型筛选，Then 列表只返回符合权限和筛选条件的结果，并展示 P0 列表字段。
- [ ] Given 用户进入龙虾详情，When 后端聚合详情数据，Then 页面展示基础配置、运行状态、模型、Skill、知识库、部署历史、基础日志、告警摘要和审计记录。
- [ ] Given 龙虾处于 `running`，When 用户执行停止或重启，Then 系统校验状态和权限，创建对应操作记录，完成后写入状态事件和审计日志。
- [ ] Given 龙虾处于 `running`，When 用户执行删除，Then 系统拒绝删除并提示必须先手动停止，且不得自动触发“停止后软删除”。
- [ ] Given 龙虾处于 `stopped`，When 用户执行删除，Then 系统执行软删除，状态流转为 `deleted`，并保留审计、状态事件和历史记录。
- [ ] Given 用户绑定 Skill，When Skill 未审核、禁用或用户无权限，Then 系统拒绝绑定并给出明确业务错误。
- [ ] Given 用户绑定知识库，When 知识库在用户数据范围内且状态可用，Then 系统保存绑定关系，并在龙虾详情中展示该知识库。
- [ ] Given 状态同步任务已执行，When 用户查看列表或详情，Then 运行态字段来自最近一次同步快照，并展示最近同步时间或可用于判断新鲜度的字段。
- [ ] Given 用户执行创建、部署、启动、停止、重启、删除、绑定或解绑操作，When 操作完成或失败，Then 系统均写入审计记录，可按龙虾实例追溯。
- [ ] Given 前端需要运行时能力，When 调用 API，Then 所有请求均走 `/api/v1/astron-claw`，浏览器网络请求中不得出现直接访问 Claw Proxy `/api/v1/bot` 的调用。
- [ ] Given 前端展示模型、令牌或沙箱相关信息，When 返回接口数据，Then 不返回完整密钥、完整令牌或可直接调用沙箱的敏感凭据。

## 风险

- 已确认本次功能建设落在当前仓库；当前可见内容以需求、模板和代码地图为主，若后续进入代码实现，需要在当前仓库补充或接入实际前后端工程结构。
- 外部实现计划范围明显大于 P0，若产品期望一次性交付批量、成本、渠道、诊断等能力，将显著扩大排期和验收面。
- Claw Proxy `/api/v1/bot` 已有可联调契约，但仍需在开发阶段核对鉴权方式、超时、重试、错误码映射和部署任务幂等性。
- P0 要求真实接入监控/日志系统，如果真实数据源交付滞后，列表、详情、日志和运行态验收会受阻；mock 数据只能用于开发联调兜底。
- RBAC、组织、席位、模型管理、Skill 管理、知识库等关联模块若尚未完成，创建和绑定流程只能先做 mock 或最小可用校验。
- 高影响操作审计和敏感字段脱敏属于合规底线，不能作为后置优化，否则会影响金融保险私有化场景验收。

## 开发任务拆分

- 前端：
  - `/agents` 列表页筛选、表格字段、空状态、错误状态和生命周期快捷操作。
  - `/agents/:id` 详情页基础信息、运行状态、模型、Skill、知识库、日志、审计分区。
  - 创建龙虾表单或弹窗，包含前置校验提示和部署进度反馈。
  - Skill/知识库绑定与解绑交互。
- 后端：
  - 龙虾主表、版本、部署任务、运行快照、绑定关系、状态事件、日志索引和审计表。
  - `/api/v1/astron-claw/agents` 列表、创建、详情、编辑预留接口。
  - 部署、启动、停止、重启、删除异步任务和状态机。
  - Claw Proxy 适配层、错误码转换、重试和失败记录。
  - RBAC、数据范围、席位、模型、Skill、知识库前置校验。
- 接口：
  - 定义列表、详情、创建、部署任务、生命周期、Skill 绑定、知识库绑定、日志、审计接口契约。
  - 统一错误码、分页、筛选、数据来源和敏感字段脱敏规则。
- 测试：
  - 覆盖创建、部署成功/失败、状态流转、权限不足、绑定失败、软删除、审计写入。
  - 覆盖前端列表筛选、详情聚合、创建表单校验、异步任务状态展示。
- 配置与运维：
  - 配置 Claw Proxy 地址、超时、重试、密钥引用、日志数据源和状态同步周期。
  - 准备开发环境 mock 数据和生产数据源切换开关。
