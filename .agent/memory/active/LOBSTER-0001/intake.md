## 初始摘要

本需求是在已有 AstronClaw ops 演示平台中建设“智能体龙虾”管理能力，不按 0-1 新项目处理，也不新增独立一级业务域。龙虾是可被创建、部署、启停、绑定 Skill/知识库、查看运行状态并接受审计的智能体实例，主入口落在 `/agents`。

本次 P0 目标是完成龙虾实例管理的最小闭环：列表查询、详情聚合、创建、部署、启动、停止、重启、删除、Skill 绑定、知识库绑定、运行状态同步、基础日志和操作审计。前端统一访问业务后端 `/api/v1/astron-claw`，业务后端通过适配层调用 Claw Proxy `/api/v1/bot`，前端不得直接访问 Claw Proxy 或持有完整模型密钥、沙箱令牌等敏感信息。

参考实现计划覆盖了批量部署、成本核算、消息渠道、问题诊断、审批流、Agent Team、备份恢复等更完整能力。结合现有模块范围知识，这些能力不进入 P0 验收，但应在数据模型、接口命名或扩展点上预留，不阻塞本次最小闭环交付。

## 已有上下文

- 相关业务知识：
  - AstronClaw 是面向企业私有化场景的智能体管理平台，已有运营管理入口 `/ops`。
  - “龙虾”是平台中的智能体实例，包含基础配置、运行状态、模型配置、Skill、知识库、日志、告警和审计记录。
  - 当前迭代已在长期知识中标记为 LOBSTER-0001，需求名称为“智能体龙虾管理模块”。
- 相关模块：
  - `/agents`：本次主入口，承载龙虾列表、详情、创建、生命周期操作。
  - `/skills`：提供已审核 Skill，供龙虾绑定和解绑。
  - `/knowledge`：提供有权限的知识库，供龙虾绑定和解绑。
  - `/models`：提供主模型、备用模型和模型策略选择。
  - `/monitoring`：提供运行状态、指标和告警来源。
  - `/ops`：承载部署、重启、日志清理、备份恢复等运行操作预留。
  - `/security`：承载审计日志、安全策略和导出审计。
  - `/org`、`/seats`：提供权限、数据范围和席位校验依据。
- 相关历史需求：
  - 暂未发现已有 active 需求档案。
- 已知约束：
  - 前端只访问业务后端 `/api/v1/astron-claw`。
  - 业务后端负责鉴权、权限、审计、持久化、任务编排和错误码转换。
  - 外部沙箱能力通过后端适配层调用 `/api/v1/bot`。
  - Claw Proxy `/api/v1/bot` 的部署、启停、重启、Skill 安装、状态查询接口契约和错误码已有可联调版本。
  - 所有高影响操作必须写审计。
  - 敏感字段不得明文返回前端。
  - 删除优先使用软删除，保留审计和历史记录；`running` 状态不允许直接删除，必须先手动停止。
  - P0 不纳入批量部署、灰度升级、复杂审批流、成本核算、完整消息渠道、Agent Team、自动诊断自愈和完整多租户资源隔离策略。
  - P0 以长期知识中的“龙虾实例管理最小闭环”为准，外部实现计划中的批量、成本、渠道、诊断等能力仅做预留。
  - 创建龙虾的最小必填字段为名称、部门、负责人、主模型、资源配置；Skill 和知识库为可选绑定项。
  - P0 要求真实接入监控/日志系统；开发环境 mock 数据只能作为联调兜底并标记数据来源，不能替代 P0 验收。
  - 本次功能建设落在当前仓库。
